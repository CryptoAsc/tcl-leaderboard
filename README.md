# TCL meets Low Priority - Leaderboard

This is a statically generated, safe, and easily hostable leaderboard for the **TCL meets Low Priority** event. No servers or backend hosting is required! It's designed to be hosted for free on GitHub Pages and uses GitHub Actions to securely update the data every 10 minutes without exposing your Faceit API key.

## Project Structure
- `docs/`: Contains the actual web application (`index.html`, `style.css`, `app.js`). This is the folder that gets served to the public via GitHub Pages.
- `docs/data/leaderboard.json`: The data file generated by our Python script. The frontend reads from this file.
- `scripts/fetch_data.py`: A Python script that securely queries Faceit, handles pagination, processes the raw data, and updates the `leaderboard.json` file.
- `.github/workflows/update-data.yml`: A GitHub Action that runs every 10 minutes to run the python script and commit any new data updates.

## 🚀 How to Deploy on GitHub

Follow these steps to deploy the application for free on GitHub Pages and set up the automated data fetching.

### 1. Push to GitHub
If you haven't already, turn this folder into a Git repository and push it to a new public repository on GitHub:
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPOSITORY_NAME.git
git push -u origin main
```

### 2. Configure GitHub Secrets (API Key)
Since we want to keep your API key secret, we will store it securely in GitHub:
1. Go to your repository on GitHub.
2. Click on **Settings** > **Secrets and variables** > **Actions** from the left sidebar.
3. Click **New repository secret**.
4. Set the **Name** to: `FACEIT_API_KEY`
5. Set the **Secret** to your API key (e.g. `f3239f46...`).
6. Click **Add secret**.

### 3. Configure GitHub Variables (Season ID)
Make it easy to switch from the testing Season 28 to Season 29 without changing any code:
1. Still in the "Secrets and variables > Actions" page, click the **Variables** tab (next to the Secrets tab).
2. Click **New repository variable**.
3. Set the **Name** to: `FACEIT_SEASON_ID`
4. Set the **Value** to your target season ID (e.g., `28` for testing, or `29` when the event starts).
5. Click **Add variable**.

### 4. Enable Read and Write Permissions for the Action
The GitHub action needs permission to commit the new `leaderboard.json` to your repository:
1. Go to **Settings** > **Actions** > **General**.
2. Scroll down to **Workflow permissions**.
3. Select **Read and write permissions**.
4. Check the box that says **Allow GitHub Actions to create and approve pull requests**.
5. Click **Save**.

### 5. Start the Automation
1. The script will automatically trigger every 10 minutes.
2. If you want to force an update immediately, go to the **Actions** tab on your GitHub repository.
3. Select the **Update Leaderboard Data** workflow on the left.
4. Click the **Run workflow** dropdown and click the green **Run workflow** button.

### 6. Enable GitHub Pages
Finally, turn on the website hosting:
1. Go to **Settings** > **Pages** (on the left menu).
2. Under "Build and deployment" > "Source", ensure **Deploy from a branch** is selected.
3. Under "Branch", select your `main` branch, and change the folder option to **`/docs`**.
4. Click **Save**.

Wait about 1–2 minutes, and look at the top of that **Pages** screen. It will give you a live URL. That is your live website!

The frontend is built with a deep "Low Priority" theme and glassmorphism. It explicitly highlights players with win streaks of 3, 4, and 5+. When a player reaches 5 wins, the UI updates with an exclusive tracker highlighting their escape!
